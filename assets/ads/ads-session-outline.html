***大衛會議廣告信息說明***


基本詞彙

廣告類型

  • 線性廣告 ：播放期間出現的廣告，需要時間在時間軸上（例如預先播放）
  • 非線性廣告 ：不會中斷播放；重疊廣告和隨附廣告為範例

廣告標準

  • VMAP：「廣告播放清單」；指定播放前、中捲和後捲 (VMAP 文件的 URL)；格式為 XML

    • DFP 呼叫 VMAP 文件「廣告規則」；使用廣告規則，您可以在伺服器上設定所有廣告，以及瞭解廣告規則的任何客戶端的行為相同；在客戶端的「壞日子」廣告設定中；邏輯移至發佈系統

  • 廣大：個別廣告詳細資料；VMAP 會指向包含廣告相關資訊的 VAST 文件

  • Chrome 開發工具中的網絡選項卡中的示例

  • 已付款：像 VAST 一樣，但互動式廣告的標準; 廣告包含測驗或愚蠢的遊戲（還沒有 BC 播放器支持）

IMA 插件信息

  • IMA 約束：2 個版本的 IMA 庫，閃存或 HTML 版本，不能同時使用

    • 如果有的話，外掛程式的預設設定會使用 Flash (因為 Flash 廣告非常普遍)
    • 在插件的配置選項中，可以將默認設置切換為HTML（ adTechOrder）
    • 桌面預設快閃記憶體，行動預設 HTML

  • IMA 外掛程式的基本實作

    • 在外掛程式中加入
    • 將 IMA 外掛程式加入播放器
    • 提供伺服器 URL 至廣告代碼
    • 範例組態

  • 動態廣告目標設定：在收集客戶資訊後完成 IMA 外掛程式的設定 (例如觀眾年齡，然後選取廣告規則)

    • 在外掛程式中加入
    • 調用 IMA 插件並在那一點上傳遞選項
      • myPlayer.ima3( { serverURL: ...} )
  • 隨身廣告 ：IMA 插件將無縫處理線性廣告和覆蓋，伴侶廣告可能很棘手，因為必須發生播放器和頁面其他部分之間的通信;;; 使用內嵌嵌入實施
    • 使用 Google 發佈商代碼 (GPT) 資料庫，並自動填滿隨附的廣告封鎖
    • 在 IMA 網站上提供良好的教程
    • 由於 iframe 無法與父頁面互動，因此對隨附廣告使用頁面內嵌實作
    • 範例：JS 和 HTML 的伴侶廣告
