    使用 Oooyala 播放器提供視頻雲端內容

    本主題說明 Oooyala 網頁和 SDK 播放程式在您的影片移轉至 Video Cloud 後，如何繼續運作，讓您的線上影片在您完成移轉到 Brightcove 播放器的同時保持正常運作。

    注意：本主題僅適用於遷移至視訊雲端的 Oyala 客戶。

    簡介

    Brightcove提供了高度可定制的網絡播放器，並且適用於Android，iOS和tvOS的本地播放器。

    但是，我們知道，取代播放器嵌入代碼和重建設備應用程序需要時間，因此 Brightcove 建立了一個適配器，可以讓您現有的 Oooyala 播放器通過從視頻雲而不是 Backlot 檢索視頻數據繼續工作：

    請注意，要測試播放適配器，您將需要使用CXM來調整計算機和設備上的DNS設置。如需詳細資訊，請參閱新增布萊特灣 DNS 伺服器。

    它是如何運作的

    所有Brightcove播放器都通過以下方式檢索視頻和播放列表數據：播放API。不過，您不必手動設置 API 請求。網絡播放器有一個目錄庫處理請求，並且設備SDK具有內置的方法。

    為了保持您的 Oooyala 播放器運作，Brightcove 建立了一個適配器，將播放器要求的視頻和播放列表數據指向播放器 API 和視頻雲中的相同視頻。

    請注意：您將無法再修改您的 Oooyala 網路播放器的外觀和感覺。目前的外觀和感覺將被保留。

    分析

    展望未來，您將使用視訊雲端分析。修復，以允許您現有的 Oooyala 播放器與您的視頻雲視頻包括發送分析數據到視頻雲分析系統。您不需要修改您的頁面或玩家。

    您可以使用標準報告和自定義報告訪問視頻雲分析數據。分析模塊或者Analytics API。看到從Analytics REST API遷移到Brightcove Analytics API如果您將通過API獲取分析數據。

    什麼可能不起作用

    如果您已經與 Oooyala 播放器整合了附加元件，它們將繼續運作。不過，如果這些附加元件取決於無法使用的視訊資料，它們將無法正常運作。見Ooyala視頻和播放列表字段到視頻雲字段的映射欲獲得更多信息。

    另外，如果您有向Ooyala API發出請求的加載項，儘管在很多情況下，這些加載項將無法正常工作一些Backlot API端點仍將運行。探索 API 將無法運作。

    分析數據將收集並發送到Brightcove Analytics系統。

    下一步

    雖然 Brightcove 致力於在您轉換到 Video Cloud 時，讓您的 Oooyala 播放器中的影片和播放清單保持正常運作，但您應該制定計畫移至 Brightcove 播放器，以充分利用他們的功能和自訂性。
