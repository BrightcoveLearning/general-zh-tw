Oooyala 到視頻雲遷移：添加 Brightcove DNS 服務器進行測試

為了測試允許您繼續使用Ooyala播放器提供視頻雲內容的播放適配器和CMS連接器，您需要在系統中添加DNS服務器，並向Brightcove提供一些信息以進行配置。本主題將引導您完成這些步驟。

注意：本主題適用只要 Ooyala 客戶正在遷移到視頻雲。

簡介

到 測試 這播放適配器這將允許您的 Ooyala 播放器從 Video Cloud 傳送內容，和/或使用內容管理系統連接器允許您使用某些操作外場API要在 Video Cloud 中管理您的媒體，必須做兩件事：

  1. 第一的：請與您的CXM聯繫以啟動該過程。您的CXM需要從您那裡獲取IP地址，並且如果您不知道該怎麼做，它將幫助您找到它們。CXM還可以幫助您完成本文檔其餘部分中必須執行的步驟。
  2. 對於將要使用播放適配器或CMS連接器的每台計算機和設備，您需要做的是添加Brightcove DNS服務器，該服務器將發送到ooyala.com資源的請求重定向到適當的Brightcove API。後續各節中詳細介紹了執行此操作的步驟。
筆記：
  • 請注意，如果您在計算機或設備上更改了DNS，前您的CXM將其IP地址列入了白名單，這可能會禁用Internet訪問。
  • 需要額外的DNS服務器僅用於測試。一旦確認可以將所需的Ooyala播放器和服務用於視頻雲數據，就可以從用於測試的計算機和設備中刪除Brightcove DNS服務器。

MacOS步驟

假設您不在VPN上，下面是在MacOS上為Mac筆記本電腦和台式機添加Brightcove DNS服務器的步驟。

  1. 打開系統偏好設置：
  2. 打開網絡首選項：
  3. 單擊先進的設置：
  4. 打開 DNS 設置：
  5. 點擊+簽名以添加 DNS 服務器：
  6. 為新的 DNS 服務器輸入以下地址：

    35.170.73.174

  7. 點擊+在搜索域下簽名以添加搜索域：
  8. 輸入以下搜索域：

    ooyala.com

  9. 點擊好的關閉高級設置。
  10. 點擊申請以保存新設置。
  11. 重新啟動 Mac 以清除 DNS 緩存。

Windows 10步驟

以下是在Windows 10筆記本電腦和台式機上添加Brightcove DNS服務器的步驟。

  1. 打開控制面板。
  2. 單擊網絡和 Internet。
  3. 單擊網絡和共享中心。
  4. 單擊左窗格中的更改適配器設置選項。
  5. 右鍵單擊連接到 Internet 的網絡接口，然後選擇屬性選項：
  6. 選擇並檢查 Internet 協議版本 4 (TCP/IPv4) 選項。
  7. 單擊屬性按鈕。
  8. 單擊使用以下 DNS 服務器地址選項。
  9. 對於首選備用 DNS 服務器地址，請輸入35.170.73.174
  10. 點擊好的。
  11. 點擊關閉
  12. 重新啟動計算機以清除 DNS 緩存。

Android步驟

由於Android版本不同，以及更改DNS服務器的方式不同（取決於您是否具有對設備的root訪問權，以及使用的是系統設置還是第三方應用程序），我們建議您與CXM聯繫以獲取幫助在Android上進行此更改。如果您想先自己嘗試，可以嘗試使用說明這裡。

特例：虛擬專用網

如果您的計算機/設備在VPN上，則上述步驟可能不起作用，因為VPN服務器可能會使用自己的DNS服務器設置來覆蓋您的DNS服務器設置。如果發生這種情況，您將需要將Brightcove DNS服務器添加到您的VPN中。實際需要什麼命令取決於平台和VPN，但是下面顯示了MacOS示例，以幫助您理解。

在VPN接口上獲取DNS條目的subKey。例如，使用Pulse Secure，它看起來像：

sudo scutil
> get subKey [127 = State:/Network/Service/net.pulsesecure.pulse.nc.main/DNS]
> d.add ServerAddresses * 35.170.73.174
> set State:/Network/Service/net.pulsesecure.pulse.nc.main/DNS
