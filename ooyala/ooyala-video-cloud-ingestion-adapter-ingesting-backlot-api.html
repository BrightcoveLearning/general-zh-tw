簡介

使用外場API以編程方式將內容上傳到 Video Cloud。通過這種方法，您可以直接將內容管理系統（CMS）或工作流與Video Cloud平台集成。

請注意，您還可以將您的集成或應用程序遷移到內容管理系統 API和動態攝取 API隨時。

您可以使用外場API以編程方式上傳和管理視頻、縮略圖和隱藏式字幕文件。

如果要使用Backlot API提取內容，請參見：

  • 視音頻資產（用於視頻和縮略圖）
  • 使用隱藏式字幕

用於上傳文件（稱為“資產”）的主要調用是 /v2/資產 路線。

筆記：不要添加帶有空值的視頻 外部識別碼，也就是說，一個 外部識別碼 沒有價值（“”) 或值 “空值”。這種空的外部ID以後無法搜索。

視音頻資產

當您將視頻上傳到Ooyala， Ooyala會使用為您的帳戶指定的設置自動對其進行轉碼，並使視頻可通過播放器播放。

請注意，音頻資產將作為沒有視頻軌道的視頻存儲在視頻雲中。

我們建議您將視頻上傳到Ooyala在H.264中，但我們也支持許多常用的編解碼器。

要創建新的視頻或音頻資產：
  1. 創建具有基本屬性的資產
  2. 從系統中檢索“上傳URL"
  3. 對於每個“上傳URL"，將適當的文件塊放入系統中
    筆記： 當你上傳每個塊時，不要簽署請求，就像其他請求一樣。只需將塊內容放入確切的URL。這些URL已經為您簽名。

獲取新創建資產的上傳URL

    [獲取]/v2/assets/ {asset_id} /uploading_urls
筆記： 當你上傳每個塊時，不要簽署請求，就像其他請求一樣。只需將塊內容放入確切的URL。這些URL已經為您簽名。

設置上傳狀態

    [PUT] /v2/assets/ {asset_id} /上傳狀態{ "status":"uploaded" }

查看視頻流

期限 流 在這種情況下，是指由系統啟動的內部到 Ooyala 進程，用於對您的視頻進行轉碼。

筆記：您無法控製或操縱這些流；它們由系統控制。您只能查看其名稱，而不能更改它們。
    [獲取]/v2/assets/ {asset_id} /流

查看視頻的源文件

    [獲取]/v2/assets/ {asset_id} /source_file_info

獲取自動生成的預覽圖像的URL

    [獲取] /v2/assets/ {asset_id} /generated_preview_images

上傳自定義預覽圖像

    [POST]/v2/assets/{asset_id}/preview_image_files <file_contents>

設置主要預覽圖像配置

將資產的主要預覽圖像的類型設置為以下之一：
  • 生成：使用自動生成的預覽圖像
  • 上傳文件：使用上傳的自定義預覽圖像
  • 遠程網址：預覽圖像的URL
    [PUT]/v2/資產/ {asset_id} /primary_preview_image { "type":"generated" | "uploaded_file" | "remote_url" }

關於元數據

一個資產最多可以包含100個名稱/值對的自定義元數據。名稱限制為128個字符，其值限制為2,048個字符。

使用隱藏式字幕

您可以通過Backlot API向視頻或遠程資產視頻添加隱藏式字幕。

字幕支持的格式

支持的格式
文件擴展名 描述
.vtt (網絡VTT/) - 這是 HTML5 視頻播放器所需的格式 WebVTT可以用於HTTP流格式（例如HLS和DASH）的嵌入式字幕和sidecar字幕。
.TTML （以前 DXFP) - 將轉換為 WebVTT TTML（定時文本標記語言文件。通常，這種類型的隱藏字幕文件將具有.dfxp（分發格式交換配置文件）擴展名。

上傳隱藏式字幕

    [PUT]/v2/資產/ {asset_id} /封閉的字幕/ {dfxp_file_content}

獲取隱藏式字幕

    [獲取]/v2/assets/ {asset_id} /closed_captions

更新隱藏式字幕

    [PUT]/v2/資產/ {asset_id} /closed_captions

刪除隱藏式字幕

    [刪除]/v2/assets/ {asset_id} /closed_captions
