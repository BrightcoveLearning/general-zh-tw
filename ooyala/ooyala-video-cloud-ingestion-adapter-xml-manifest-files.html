Oooyal-視訊雲端擷取適配器：XML清單文件

您可以繼續將FTP，Aspera或MRSS與XML清單一起使用，以將視頻攝取到Video Cloud。本主題說明XML的結構。
注意：本主題適用只要 Ooyala 客戶遷移到視頻雲。

簡介

攝取視頻時，可以使用XML清單文件來指定其他元數據（包括視頻標題，說明，飛行時間，標籤和自定義元數據）以及縮略圖和隱藏字幕文件。通過FTP，Aspera或MRSS提取內容時，可以使用XML清單文件。

筆記：本主題適用於 OIS v2.5。

元數據的處理

如果包括metadata攝取視頻時，它會以以下兩種方式之一映射到“視頻雲"視頻字段：

  1. 默認情況下， 每個metadata鍵/值對被添加為 標籤 形式：meta::key::value .例如，<ooyala:metadata name="category">cooking</ooyala:metadata>會產生標籤meta::category::cooking
  2. 如果您願意使用custom_fields為了存儲元數據，Brightcove支持人員可以更改您帳戶上的設置來做到這一點。請注意以下幾點：
    • 您將需要使用與元數據鍵相對應的名稱來創建自定義字段。
    • 默認情況下，自定義字段的數量限制為10，因此，如果您需要更多的自定義字段來容納元數據，則需要向支持部門請求其他自定義字段。
    • 如果您的帳戶中不存在與每個元數據項相對應的自定義字段，則整個元數據操作將失敗。

XML清單元素

您可以在XML清單文件中使用以下元素。

XML清單元素
XML標籤 描述 必需/可選
<channel> 包含一項或多項 <item> 元素。 必需的
<item> 表示要提取的單個視頻。您可以指定多個視頻 (<item> 標籤）在 XML 清單文件中。包含此表中的其餘標籤。 必需的
<media:content> 指定要提取的視頻。以下之一：
  • 您上傳的視頻的文件名。
  • 通過MRSS進行攝取（請參見從遠程託管的MRSS Feed提取），即視頻所在的網址。URL 可以包括 http:// 或者 https://。
對於遠程資產，請使用 <ooyala:remoteasset> 反而。		 必需（遠程資產除外）
<media:title> 視頻的名稱或標題。 可選的
<media:thumbnail> 指定縮略圖（預覽圖像）。以下之一：
  • 您上傳的縮略圖的文件名。
  • 縮略圖所在的網址。URL 可以包括 http:// 或者 https://。
 可選的
<media:subTitle> 指定隱藏式字幕文件（DFXP，WebVTT和SRT文件支持）。看攝取隱藏式字幕文件。屬性：
  • href （必需）- 以下之一：
    • 您上傳的隱藏式字幕文件的文件名。範例： 標題-1.dfxp
    • 隱藏式字幕文件所在的URL。URL 可以包括 http:// 或者 https://。
    朗 （可選）- 隱藏式字幕文件的語言。兩個字母的代碼。看支持的隱藏式字幕。
  • 幀率 （可選）-隱藏式字幕文件的幀頻。單位為每秒幀數（FPS）。默認值為30。較早的隱藏式字幕文件可能是24。
筆記：如果指定，則您有責任為 朗 和 幀率。有效值將添加到DFXP文件中。不符合的值將被忽略。無論哪種方式，隱藏式字幕文件都被添加到Backlot中並鏈接到資產。
 可選的
<media:description> 影片說明。 可選的
<link> 嵌入視頻的永久URL。映射到創建的資產的 託管_at 財產。 可選的
<dcterms:valid> 飛行時間代表可以播放資產的開始時間和結束時間。範例： 
  <dcterms:valid>start=2011-06-28T15:15:00-07:00; end=2022-12-01T16:00:00-07:00;scheme=W3C-DTF </dcterms:valid>
 可選的
<ooyala:labels> 一個或多個標籤，以逗號分隔。範例： /運動，/運動/慢跑。有關背景，請參見標籤。 可選的
<media:keywords> 定義標籤的另一種方式（不帶正斜杠）。以逗號分隔的內容將作為標籤添加到視頻中。僅當不存在具有相同名稱的標籤時，才會創建一個新標籤。

例如，對於以下 <media:keywords> 標籤，標籤 標籤 1 和 標籤2 將被創建並作為標籤添加到視頻中。一個名為 父標籤 也將被創建，帶有標籤 子標籤 嵌套在裡面。

  <media:keywords>label1, label2, parentlabel/childlabel</media:keywords>
筆記：的內容 <media:keywords> 被忽略時 <ooyala:labels> 標記包含在 XML 清單文件或 MRSS 中。
 可選的
<ooyala:embedcode> Video Cloud視頻ID。 可選（如果您要更新視頻而不是添加新視頻，則為必填項）
<ooyala:profileguid> 處理配置文件的ID，它是一組編碼，這些編碼定義了在轉碼過程中主視頻必須轉換成的格式。有關詳細信息，請聯繫您的Ooyala支持代表。 可選的
<ooyala:metadata> 視頻的自定義元數據。您可以為每種類型的元數據都有一個元素。見元數據的處理部分，了解有關如何操作的詳細信息metadata作為特殊標籤添加到視頻中。例子：<ooyala:metadata name="category">cooking</ooyala:metadata> 可選的
<ooyala:remoteasset> 指定要提取的遠程資產。遠程資產是託管在Backlot之外的其他地方的視頻。指定以下屬性：
  • 持續時間InMs：媒體持續時間（以毫秒為單位）
  • 網址：遠程資產的URL。URL 可以包括 http:// 或者 https://。
此設置映射到 託管_at 遠程資產的屬性。範例： <ooyala:remoteasset durationInMs='5000' url="http://your_website/your_remote_assets.mp4" />
筆記：創建遠程資產後，其傳播到各種CDN的時間可能會延遲60秒或更長時間。如果您在創建遠程資產後不久請求遠程資產，結果將被CDN緩存，這可能需要幾分鐘的時間才能清除。最佳實踐：創建後，等待 30 或 60 秒，使用外場API [獲取] /v2/assets/資產 ID 路由，並在檢索遠程資產的嵌入代碼（內容 ID 或資產 ID）後，然後繼續嵌入資產。
 必需（僅限遠程資產）
<ooyala:profileguid> 指定用於處理視頻的視頻雲提取配置文件。如果不想使用帳戶默認配置文件，則必須指定要使用的默認配置文件。看管理攝取配置文件。 可選的

範例

XML清單文件示例（OIS v2.5）

  <?xml version="1.0" encoding="utf-8"?> <rss version="2.0" xmlns:media="http://search.yahoo.com/mrss/" xmlns:dcterms="http://purl.org/dc/terms/" xmlns:fh="http://purl.org/syndication/history/1.0" xmlns:ooyala="http://www.ooyala.com/mrss/"> <channel> <item> <media:content url="test.mp4" ></media:content> <media:subTitle href="my_caption.dfxp"></media:subTitle> <media:thumbnail url="my_thumbnail.jpg" filesize="35712"></media:thumbnail> <media:title>Test XML</media:title> <media:description>This is an example</media:description> <ooyala:labels>/test,/example,/xml</ooyala:labels> <dcterms:valid>start=2016-01-01T15:15:00-07:00; end=2022-01-01T16:00:00-07:00</dcterms:valid> <ooyala:profileguid>57d5a5a10f654fe79ef954dc8d29a108</ooyala:profileguid> </item> </channel> </rss>

一步步

創建一個XML清單文件

要創建XML元數據文件：

  1. 上載您的視頻和其他內容（縮略圖，隱藏式字幕文件等）。
  2. 打開一個文本編輯器，最好是一個驗證XML的文本編輯器。
  3. 添加所需的XML標頭。例如： 
      <?xml version="1.0" encoding="utf-8"?> <rss version="2.0" xmlns:media="http://search.yahoo.com/mrss/" xmlns:dcterms="http://purl.org/dc/terms/" xmlns:fh="http://purl.org/syndication/history/1.0" xmlns:ooyala="http://www.ooyala.com/mrss/">
  4. 創建一個 渠道 容器。
  5. 創建一個 物品 每個視頻的條目，包含上表中所需的元素。

    在以下示例中，XML指定了兩個視頻的元數據。對於第一個視頻，文件和縮略圖位於上傳服務器上，並從該服務器中拉出。對於第二個視頻，文件和縮略圖已經上傳到Ooyala服務器。第二個視頻也有飛行時間。

      <?xml version="1.0" encoding="utf-8"?> <rss version="2.0" xmlns:media="http://search.yahoo.com/mrss/" xmlns:dcterms="http://purl.org/dc/terms/" xmlns:fh="http://purl.org/syndication/history/1.0" xmlns:ooyala="http://www.ooyala.com/mrss/"> <channel> <item> <media:content url="http://mysite.com/upload/lacrosse_70.mov" /> <media:thumbnail url="http://mysite.com/upload/lacrosse_70_previewimage.jpg" /> <media:title>Sports that Really Exist:Lacrosse</media:title> <media:description>My description of my video</media:description> <ooyala:labels>/sports/lacrosse,/hobbies/lacrosse</ooyala:labels> <link>http://mysite.com/videos/sports/real_sports.html</link> <ooyala:metadata name="video_ID">70</ooyala:metadata> <ooyala:metadata name="season_number">1</ooyala:metadata> </item> <item> <media:content url="curling_71.mov" /> <media:thumbnail url="curling_71_previewimage.jpg" /> <media:title>Sports that Really Exist:Curling</media:title> <media:description>My description of my video</media:description> <ooyala:labels>/sports/curling,/hobbies/curling</ooyala:labels> <link>http://mysite.com/videos/sports/real_sports.html</link> <ooyala:metadata name="video_ID">71</ooyala:metadata> <ooyala:metadata name="season_number">1</ooyala:metadata> <!-- set flight times --> <dcterms:valid>start=2011-06-28T15:15:00-07:00; end=2022-12-01T16:00:00-07:00;scheme=W3C-DTF </dcterms:valid> </item> </channel> </rss>
  6. 將文件另存為XML。確保指定唯一名稱。如果您指定兩個具有相同名稱的文件並在處理另一個文件時上傳一個文件，則第二個文件可能會被忽略。
  7. 使用您選擇的上載方法，在上載視頻時上載XML清單文件。見的FTP或者阿斯佩拉。
