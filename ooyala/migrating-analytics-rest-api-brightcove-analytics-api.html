簡介

視頻雲分析 API與 Ooyala Analytics REST API 非常相似。

另請注意，Video Cloud Studio中的Google Analytics（分析）模塊包括一個功能強大的自定義報告生成器在許多情況下可能根本不需要使用該API。

以下各節對應於兩個API的不同方面：

  • 驗證
  • 報告書
  • 參數
  • 方面
  • 篩選器
  • 指標

驗證

Brightcove API請求的身份驗證與Ooyala REST API的身份驗證稍有不同。看Brightcove API請求的身份驗證有關詳細信息。

報告

單曲/reports Ooyala Analytics REST API 的端點與 Brightcove 非常接近分析 API/data端點。主要區別在於Ooyala API允許兩者GET和POST，後者用於在請求正文中傳遞一些數據，而不是傳遞URL參數，從而允許使用更長的數據字符串。

方面

篩選器

過濾器在兩個API中的工作原理幾乎相同，分別指定為：

      
        // Ooyala
        filter={dimension1}=={value1};{dimension2}=={value2}
      
        // Brightcove
        where={dimension1}=={value1};{dimension2}=={value2}

測量結果

這metrics（對於Brightcove API：fields報告中返回的）取決於Brightcove Analytics API要求的尺寸。您可以使用內嵌的工具尺寸和領域查看可以針對特定維度或維度組合返回哪些字段。

請注意，雖然Ooyala API如果未指定所有指標，則返回所有指標，但Brightcove API僅返回一小部分默認指標，這些指標會因維度（組合）而異，除非您添加了fields在請求中。

Brightcove字段包含所有Ooyala指標的等效項，但以下各項除外：

  • uniq_plays_requested
  • uniq_displays
  • uniq_video_starts

明亮灣分析 API還可以返回 Ooyala 分析中不可用的其他指標的數量。
