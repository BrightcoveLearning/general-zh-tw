Oooyala 視訊雲端擷取配接器：支援的內嵌格式

本主題說明瞭如何使用CSV清單通過以下方式提取文件： Ooyala-視頻雲提取連接器
注意：本主題適用只要 Ooyala 客戶遷移到視頻雲。

內嵌格式

下表列出支援的內嵌格式。

類型 格式
視訊轉碼器
  • 受到推崇的：H.264
  • 其他支持的格式： 看支持的視頻和音頻格式，編解碼器和容器
音訊轉碼器
  • 受到推崇的：AAC
  • 其他支援的格式：請參閱支援的視訊和音訊格式、轉碼器和容器

檔案名稱

在擷取程序期間，必須將來源檔案名稱作為 API 要求 URL 的一部分傳遞。因此，名稱必須與 URI 相容。請避免在所有視訊和資產的來源檔案名稱中使用下列項目：

  • 空間： 代替my video.mov， 用my-video.mov或者my_video.mov
  • 特殊的角色： 做不是在源文件名中使用以下字符："!", "@", "#", "$", "%", "^", "&", "*", "(", ")", "?", "/", "\", "{", "}", "[", "]"

支援的檔案格式

視訊檔案

自從亮灣支持這麼多的視頻輸入類型，其實說一下我們不支持的格式更有幫助。目前，只有少數。下面的列表顯示了格式不支援視訊雲端。

  • Apple Intermediate
  • HDV 720p60
  • Go2Meeting3 (G2M3)
  • Avid Meridien Uncompressed
  • Windows Media Audio Lossless
  • CineForm -部分支持；有些變體可能來自用於對源進行編碼的軟件，也可能來自操作系統，從而使該源無法被Video Cloud使用。我們仍在試圖找出導致問題的確切因素。
  • SWF

特殊案例

  • 筆記：AVdh (DNxHQ)也不適用於422 個 10 位和 444 個 12 位源（僅限） - 這是一個錯誤，修復後將刪除此註釋。
  • 支持使用KMS加密的內容，但要求您對對象URL進行預簽名，例如此處概述的內容： https://aws.amazon.com/blogs/developer/generating-amazon-s3-pre-signed-urls-with-sse-kms-part-2/。

    實作步驟：

    1. 建立 KMS 管理的加密儲存貯體
    2. 新增視訊雲端/Zencoder 擷取政策
    3. 使用 KMS 金鑰建立預先簽署的 URL
    4. 擷取視訊時，請使用此 URL

除了上面列出的內容之外，我們相信我們支援幾乎所有已知的視訊格式。如果您有其他視頻格式的問題，請接觸亮灣支持並且還讓亮灣學習服務知道。

音訊檔案

大部分的格式都受到支援，除非它們取得超過 8 個頻道的音訊。音訊必須設定為單一多聲道音軌，或是多個單聲道音軌。)

筆記：對於視頻云攝取，如果您的源具有多通道音頻，則需要支持創建自定義音頻再現以正確映射通道。

隱藏式字幕檔案

攝取隱藏式字幕文件。請注意，像布萊特灣和奧亞拉球員這樣的 HTML5 網絡播放器需要 WebVTT 字幕。如果你攝取 DFXP或SRT字幕，它們將自動轉換為WebVTT。對於遠程資產標題，您必須使用WebVTT格式。

縮圖和海報檔案

JPG、JPEG、PNG 和 GIF

最佳實務

下列建議將有助於確保您的來源檔案已準備好提供最佳結果。

逐行掃描，逐行掃描

來源視訊應先進行漸進式掃描 (漸進式、非交錯式、解交錯)，然後再上傳至 Video Cloud。雖然 Oooyala 可以在處理來源視訊時解除交錯內容，但我們建議，如果您有資源，請在攝入之前執行此操作。這可讓您在編輯工作流程的品質控制階段確認視訊來源資產的品質。
