Ooyala 視訊雲端擷取配接器擷取：從遠端託管的 MRSS 摘要擷取

您可以繼續通過MRSS Feed將視頻和資產攝取到Video Cloud中。攝取適配器。
注意：本主題適用只要 Ooyala 客戶遷移到視頻雲。

簡介

您可以使用媒體 RSS (MRSS) 讓 Oyala 從您發佈的 MRSS 摘要中提取您自己的系統中的內容。Video Cloud從MRSS feed中提取元數據，並創建遠程資產或下載（從託管它們的位置）並處理視頻和任何相關文件（縮略圖，隱藏式字幕文件等）。

筆記：本主題適用於 OIS v2.5。

MRSS 內嵌概觀

關於 MRSS

媒體 RSS（MRSS）是 RSS 的擴展，允許您在 RSS 提要聯合多媒體文件。有關技術細節，請參見媒體RSS規範。

設定 MRSS

請聯絡您的客戶經理，並指出您想要使用 MRSS。對於您要提取的每個視頻，必須向Ooyala提供有效的內容URL。URL 可以包括 http:// 或者 https://。

使用 MRSS

若要使用 MRSS，您可以在 Web 伺服器上裝載中繼資料和/或視訊和其他檔案 (縮圖、隱藏式字幕)，並建立參照 該內容的 XML 資訊清單檔案。

使用 MRSS 僅提取中繼資料

  1. 如中所述創建XML清單文件XML清單文件。您必須使用 ooyala：遠程資產 僅用於元數據攝取的標記。
  2. 將 XML 資訊清單檔案裝載在您提供的其中一個 MRSS 位置上。確保文件名和路徑與您提供給Ooyala的路徑之一完全匹配。

使用MRSS提取視頻，縮略圖和隱藏式字幕文件

  1. 在網頁伺服器上裝載一或多個檔案 (影片、縮圖和隱藏式字幕檔案)。
  2. 創建一個XML清單文件（請參閱XML清單文件）。請確定檔案中的項目參考伺服器上的檔案。特別是，價值 <media:content> 元素的 href 屬性 必須 是服務器上文件的 HTTP URL，如下例所示： 
    <media:content url="http://mysite.com/upload/lacrosse_70.mov" />
  3. 將 XML 資訊清單檔案裝載在您提供的其中一個 MRSS 位置上。確保文件名和路徑與您提供給Ooyala的路徑之一完全匹配。

    筆記：不幸的是，「用戶端記錄」的概念並不適用於 MRSS 饋送式處理。驗證您的 Feed 是否成功處理的最佳方法是使用 Video Cloud Studio UI 或外場API /v2/資產 要求。

在處理完視頻和其他任何相關文件後，要提取更多文件，您只需在約定的位置託管另一個MRSS文件。
