注意：本主題適用只要 Ooyala 客戶遷移到視頻雲。

新的視訊雲端選項

視訊雲端有三個標準選項可用於擷取視訊：

  • Studio中的“上傳"模塊
  • 使用Dynamic Ingest API進行基於請求的提取
  • 使用Dynamic Ingest API上傳源文件

或者使用 Oooyala 攝取方法

作為Video Cloud客戶，您可以使用上面列出的任何選項，但是Brightcove還為Ooyala攝取系統構建了適配器，以將視頻定向到Video Cloud攝取系統，因此您可以繼續使用Ooyala攝取服務（OIS）使用以下方法將視頻發送到Video Cloud：

  • FTPS
  • 阿斯佩拉
  • 遠程託管MRSS提要
  • Backlot API

所有現有的資訊清單選項將繼續運作。繼續使用 Oooyala 的身份驗證憑證，和適配器會自動將這些認證轉換為必要的視頻雲 API 的認證：

以 API 為基礎的擷取
如果您的擷取工作流程包括直接透過擷取 API 擷取新內容：當您的用戶端流量導向至視訊雲端時，您將使用現有的 Oooyala API 金鑰和秘密，將 API 呼叫目前的 Oooyala API 端點。
基於 FTP/ 亞斯佩拉的攝取
確保將視頻可以“僅上傳"用戶帳戶密碼重置為與Backlot中不同的密碼。如果您在 Backlot 中的「僅上傳」帳戶支援影片和中繼資料，則影片雲端的「僅上傳」帳戶也會這樣做。您可以繼續連線至相同的 FTP/ASpera 端點，並使用新的 Video Cloud「僅上傳」使用者帳戶憑證來上傳檔案。一旦檔案已上傳並從上傳資料夾中消失，請登入 Video Cloud 並確保您的影片已新增。如果您要上傳 XML 或 CSV 中繼資料檔案以及視訊，並且想要鎖定特定的擷取描述檔，則必須在設定檔元素下的 XML 或 CSV 欄位中指定 Video Cloud 設定檔名稱。看管理攝取配置文件。
以 MRSS 為基礎的擷取
請與您的帳戶團隊分享您的 MRSS 摘要，他們可以啟用它。因為我們將無法驗證 MRSS 摘要中的哪些影片先前已被內嵌到 Backlot，因此請確保 MRSS 摘要中僅提供新影片。如果舊影片在動態消息中可用，這些影片也會內嵌在 Video Cloud 中產生重複的內容。一旦 MRSS 新聞稿到位，您就可以登入視訊雲端來驗證內容是否正在擷取。
通過布萊特灣視頻雲端工作室攝取
Backlot的一個重要區別是您將選擇用於攝取內容的攝取配置文件，請確保在Video Cloud中選擇最符合您要求的攝取配置文件，您可以在以下位置找到有關Video Cloud攝取配置文件的有用信息管理攝取配置文件。
