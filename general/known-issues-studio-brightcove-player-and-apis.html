工作室

使用Chrome的翻譯功能會破壞某些Studio功能
使用Chrome的翻譯功能翻譯Studio模塊會導致某些功能中斷。

媒體模塊

預覽播放器
只能播放HLS的實時遠程資產視頻不能在預覽播放器中播放。
包含雙字節字符的源文件名
在媒體模塊中重新編碼後，包含雙字節字符的源文件名會出現亂碼。
媒體分享

當共享大量視頻時，Brightcove建議一次最多共享兩頁視頻。一次共享大量視頻可能會導致超時錯誤。

不支持向已包含字幕的共享視頻添加字幕。

對於通過剪輯實時流創建的視頻，媒體共享將失敗。

預定影片
由於Playback API和商品目錄最多可緩存視頻10-15分鐘，因此要求播放器計劃在接下來的幾分鐘（最多20分鐘）內可用的視頻的播放器可能無法獲取可播放的視頻，直到緩存刷新為止。

播放器模塊

預載設定
由於Internet Explorer中的錯誤，使用IE時可能會忽略預加載設置。

即時視訊

Brightcove Player為完成的現場活動計算視頻觀看次數
每次加載遠程資產時，Brightcove播放器都會佔用video_view，即使這是一個已完成的直播活動，其HLS播放列表為空或不再可獲取。解決方法：停用，取消預定或刪除視頻。
 

分析

由於廣告屏蔽，分析數據可能不准確
由於廣告攔截器（例如UBlock）阻止了客戶端計算機上分析數據的提交，因此分析數據（通過UI和API獲得的數據）可能不准確。
自定義報告顯示0個字節，已傳送給某些視頻
使用動態投放攝取配置文件攝取的視頻將在自定義報告中顯示0字節交付的視頻。
使用多個過濾器的自定義報告
使用多個過濾器時，將使用 AND 操作。例如，如果您添加兩個過濾器，影片標記和播放器，僅具有指定標籤的視頻和在指定播放器中觀看的內容將被退回。
使用“按視頻標籤過濾"功能的自定義報告
創建自定義報告時，使用按視頻標籤過濾功能會受到限制。它最多只考慮您的媒體庫中帶有該特定標籤的2,000個視頻。如果帶有特定標籤的視頻數量大於該數量，我們建議您獲取一個包含標籤的未過濾報告，並使用電子表格軟件在本地進行過濾。
“其他/第三方"出現在效果報告中
當不是您的視頻雲庫的視頻在視頻雲播放器（例如，遠程資產）中播放時，其他/第三方將顯示沒有關聯的視頻ID。
更新的視頻名稱未反映在Google Analytics（分析）中
Analytics（分析）唯一了解視頻標題的時間是播放器將其作為播放信標的一部分發送到Analytics（分析）時。如果更改了視頻標題，則在進行視頻播放之前，分析將不會擁有新的視頻標題。這可能會導致視頻名稱被更新並且如果未錄製任何回放的情況，則舊的視頻名稱將反映在Google Analytics（分析）報告中。2013年3月之前的時間段數據經常受此問題困擾，並且顯示的元數據僅為數字。例如，視頻標題可能顯示為1230123012。這是因為這段時間的數據是在舊的分析系統中捕獲的。
顯示的視頻名稱和播放器名稱在Google Analytics（分析）用戶界面中
對於2013年10月，元數據（視頻名稱和播放器名稱）可能看起來像一堆?????。此問題影響到元數據具有雙字節字符並在2013年10月具有視頻觀看次數的發布商。此問題與我們捕獲元數據以在Analytics（分析）UI中顯示的方式有關。從2013年11月開始，Analytics（分析）系統將採取其他保護措施，以防止隨事件指標一起發送的元數據被破壞。
報告時區
報告時區是用來計算報告資料的日期界限。變更報告時區設定只會影響未來的資料，而且不會追溯套用變更。如果您變更此設定，您可能會在查看套用變更的日期邊界時，看到資料中的平坦位置或尖峰。此設定的更新可能不會立即生效，而分析資料可能會繼續參照舊的設定，直到系統處理變更為止。
參與度數據按每日粒度存儲
互動度數據是使用UTC時間以每日粒度存儲的，有時可能會在“互動度報告"的“x％受眾"表中包含2天的數據。
Internet Explorer支持
Analytics（分析）模塊需要Internet Explorer 11或更高版本。
設備製造商的大量“其他"結果
在某些情況下，設備製造商可能會看到大量“其他"結果。當數據收集器無法識別user_agent與分析數據一起交付給它。我們已經確定了這個最常見的原因是由 Instagram 創建的自定義 user_agent 字符串，但可能還有其他人。

畫廊

與水平播放列表IPX一起使用時，聊天記錄登錄失敗
Chatroll在從iframe設置cookie時遇到問題。Safari僅允許原始站點（在這種情況下為chatroll.com）具有cookie的用戶從iFrame中存儲它們。如果用戶從未訪問過它，則IPX中的嵌入式聊天記錄無法設置cookie。這不是Brightcove問題，這是Safari和Chatroll的問題。

受眾

觀眾活動顯示觀看百分比：0和觀看時間：0
有時，互動事件是在沒有視頻播放的情況下或在視頻發送其第一個信標之前觸發的。這可能會導致錄製的觀看事件具有互動性，但沒有視頻活動（觀看0％，觀看0秒）。
Internet Explorer支持
觀眾潛在客戶表單需要Internet Explorer 11或更高版本。

社交

Ghostery瀏覽器插件可能會干擾社交模塊

Ghostery瀏覽器插件可能會干擾社交模塊，從而阻止其正確加載。

解決方法：將Studio添加到Ghostery的白名單中。

上傳模塊/動態提取

如果設備上的時鐘與實際時間不匹配，則無法通過上傳模塊上傳
授權上傳一定時間。如果您的時鐘時間與實際時間相差超過15分鐘，則上傳時會收到錯誤消息。
通過動態提取進行重新轉碼會更新視頻激活日期
當您使用動態提取對視頻進行重新轉碼時，視頻的激活日期將更新為當前日期。如果您使用按激活日期排序的智能播放列表，這將影響播放列表中視頻的順序。
檔案名稱
視訊檔案名稱 (包括副檔名) 不得超過 120 個單位元組 (60 個雙位元組) 字元。如果這樣做，視訊將會成功攝入，但您將無法在稍後重新轉碼。

布萊特灣玩家

檢查Brightcove Player發行說明查看最近發布的版本是否已糾正了過去的已知問題。

Web Player的可訪問性：無法調整視頻位置
Brightcove Player 進度控制和音量滑塊無法通過本機手勢進行調整 iOS 上的旁白 和 Android 上的 TalkBack。這是由於平台限制。而是，用戶必須直接與元素進行交互。
  • 在iOS上，用戶必須雙擊以選擇進度條或音量滑塊，然後雙擊並按住將允許用戶調整時間或音量。
  • 在Android上，用兩根手指雙擊將允許用戶將進度或音量調整到兩根手指雙擊的位置。
元素與id調成default
如果有一個元素存在，則已知Brightcove Player的行為較差id="default"在頁面上。博客文章DOM：元素ID是全局變量解釋了為什麼這是一個問題。

沒有海報或縮略圖的視頻
沒有縮略圖/海報圖像的視頻在加載時可能會導致播放器中出現錯誤。這以可能會使播放器崩潰的方式影響移動SDK播放器。
每當播放暫停時，Internet Explorer始終將其播放速率重置為1.0。
這將不再是Safari 14（Big Sur版本）中的問題。
Safari上的自動前進播放列表
對於Safari，如果播放列表為可見的，並且玩家擁有pre-load 設置為none ，下一個視頻的自動播放將自動失敗。
自動打開字幕在Safari中不起作用
觀看者仍然可以手動打開字幕。
上下文菜單
右鍵單擊靠近播放器右側或底部邊緣的視頻時，播放器上下文菜單顯示在屏幕外。
在頁面上使用具有相同player_id的大小不同的多個玩家

如果多個玩家在一頁上具有相同的ID，並在其中指定了不同的大小<video>標記使用width和height屬性，最後一個播放器的播放器CSS會應用於所有播放器。

解決方法

  • 如果您希望他們使用不同的玩家ID或樣式，請使用具有不同玩家ID的玩家。
  • 為使用CSS提供播放器尺寸的播放器定義一個類，然後將這些類添加到不同播放器的視頻代碼中
FastClick.js導致錯誤的事件處理
使用FastClick.js會導致我們的自定義控件欄中的事件處理不正確。這可能導致無法在移動設備上使用我們的控制欄。
視頻在Chrome和Firefox上顯示為灰色

使用Chrome和Firefox時，Brightcove Player中的視頻可能顯示為灰色。這可能是由於硬件加速和/或NVIDIA驅動程序設置所致。

解決方法：打開NVIDIA控制面板。在下面視頻選擇調整視頻顏色設置。在下面您如何進行色彩調整選擇使用NVIDIA設置。在下面先進的確保動態範圍是全（0-255）不是限量（16-235）。動態對比度增強應該取消選中。此問題已在Google Chrome瀏覽器幫助論壇。

使用模擬器

無論您是使用Chrome設備模式還是其他仿真器來測試視頻播放，請注意，仿真器無法準確表示實際設備的性能。雖然可以在開發過程中使用仿真器進行初始測試，但最好的方法是使用真實的設備來獲得準確的結果。

在Chrome設備模式下測試播放時，您可能會看到以下消息：“在設備模式下使用Chrome只會在Chrome中呈現該設備的視口和用戶代理字符串，這不能準確表示實際設備的性能。"

使用data-setup
你不應該使用data-setup 與Brightcove Player。您可能會看到使用data-setup API文檔中的內容，但這是因為該文檔是直接從Video.js播放器源代碼生成的，因此您可以將該屬性與video 帶有純Video.js的代碼。該屬性將配置信息發送到播放器，但是Brightcove Player使用不同的方法來執行此任務，這使得data-setup 不可靠。
協議感知源選擇和DASH
使用DASH內容時，協議可用的源選擇不可用。僅適用於HLS和MP4內容
在Safari中使用iframe播放器實現時引發控制台錯誤
在Safari中使用Brightcove Player的iframe實現時，您將在控制台中看到以下錯誤消息：阻止了源為“http://players.brightcove.net"的幀訪問源為“#DOMAIN NAME＃"的幀。協議，域和端口必須匹配。報告的錯誤不會影響播放。
Safari的字幕菜單中顯示“未知"。
這是Safari / Apple的限制。較新版本的Brightcove Player使用Safari內置的本機字幕功能，這是標準行為。這是由Apple記錄的： https://developer.apple.com/library/ios/qa/qa1801/_index.html
非Flash環境中的全屏

在禁用播放器Flash或禁用基於Flash的HLS的環境中，使用style 的屬性video 標籤，在某些瀏覽器中可能無法全屏查看。

解決方法：刪除style 來自的屬性video 標記，而是在頁面樣式表中創建規則，如下所示：

  .video-js{
  width:640px;
  height:360px;
  }
社交分享
在台式機上，社交共享按鈕在廣告期間將不可見。
HTTPS

在大多數桌面瀏覽器上，當清單和視頻段都通過HTTPS連接提供時，Brightcove Player僅在HTTPS網站上播放HLS。這是由於最近對幾種瀏覽器進行的更改所致，這些更改更嚴格地限制了非SSL內容。這會影響台式計算機上的Chrome，Firefox和Internet Explorer用戶。它不會影響Safari用戶或移動瀏覽器，也不會影響MP4再現的播放。

我們正在努力解決視頻雲託管資產的這一限制；如果您管理自己的CDN和轉碼（遠程資產），則必須配置CDN以支持清單和視頻內容的HTTPS傳遞。

全屏顯示

在支持全屏API的較新瀏覽器中，有必要應用頁內CSS規則，以確保在切換到全屏時將播放器縮放到100％。否則，播放器將以全屏顯示的原始尺寸顯示。有關詳細信息，請參見全屏顯示主題調整播放器大小文檔。

對於IE10及更早版本，不支持全屏API，將打開一個新窗口，但播放器的大小將無法填充該窗口。這是因為尚未應用任何樣式來調整播放器的大小。由於無法滾動窗口，因此您可能只會看到網站的一部分，根本沒有播放器。

HapYak章節將多個視頻發佈在單個頁面上

當在同一頁面上發布包含HapYak章節的多個視頻時，可能會顯示錯誤的章節。

解決方法

使用此腳本： https://github.com/brightcove/gallery-in-page-snippets/blob/master/snippets/hapyakFix.js。

Akamai HD和HDS
Brightcove Player不支持通過Akamai HD或HDS交付方式進行視頻交付。
錯誤： d.off不是函數

在播放器中在不同格式（例如MP4和HLS）之間進行切換時，可能會發生由已知錯誤引起的此錯誤。在修復該錯誤之前，您可以簡單地重試導致該問題的代碼。以下代碼是糾正應用程序中錯誤的示例：

  try {
  myPlayer.catalog.load(video);
  } catch (e) {
  myPlayer.catalog.load(video);
  }
  myPlayer.play();
HLS視頻時長
控制欄中顯示的持續時間可能會與初始顯示有所不同。一旦所有的HLS段被加載，持續時間就可以更新。
需求JS
如果您使用的是RequireJS，則可能必須使用bc() 實例化播放器的方法。是否需要使用該方法將取決於玩家資產的加載方式/時間。見RequireJS和Brightcove Player文檔以獲取更多信息。
Referrer_url與description_url
這referrer_url 該值在iOS和Android設備之間可能具有不同的值。因此，建議使用description_url 取而代之。該值在所有平台和設備上都是一致的。
Safari上的循環屬性
這環形屬性在Safari上無法正常工作。您可以使用 Brightcove Player示例：創建視頻循環文檔。
Safari上的字幕
由於Safari使用本機功能來設置字幕樣式，因此字幕設置菜單中的菜單選項不可用抄送 Safari瀏覽器（iOS和macOS）上的菜單。
使用名為global的HTML元素ID
您不應將高級（頁內嵌入）播放器實現代碼嵌套在父級中<div> 用一個標記id 賦值global 。這會導致Brightcove Player出現問題。
播放器版本5圖標問題
Brightcove Player 5版的章節和字幕均使用相同的圖標。由於版本5處於維護模式，並且這是一個修飾問題，因此是否可以解決該問題是令人懷疑的。

安卓

  • Brightcove Player進度控制和音量滑塊無法通過Android上“話語提示"的本機手勢進行調整。這是由於平台限制。而是，用戶必須直接與元素進行交互。在Android上，用兩根手指雙擊將允許用戶將進度或音量調整到兩根手指雙擊的位置。
  • 在設備上，進度條可能與HLS視頻不同步，因為HLS在Android上無法很好地運行。視頻的總時長也可能會錯誤地顯示為0:01。
  • 存在多個可訪問性問題 本國的 適用於所有 Android 版本的手機和平板電腦上的瀏覽器。對講不為任何播放器控件提供音頻和振動反饋。（這個問題確實 不 適用於Android設備上的插件Chrome瀏覽器，僅適用於本機瀏覽器。
  • 在設備上，錯誤消息的點擊事件不會冒泡到父視頻元素。這意味著一旦出現錯誤消息，您將無法關閉。
  • 在Android設備上，在導航欄中瀏覽播放器菜單（如標題或質量選擇）時，長按菜單項後，菜單可能會“卡在"打開狀態。這是因為Chrome瀏覽器添加了:hover 偽類。再次長按播放器上的其他位置通常會關閉菜單。

  • 切換到另一個應用程序時，iOS中會發生播放器錯誤。在Safari / Chrome的iOS設備上播放Brightcove Player中的視頻時，播放器在切換到另一個應用程序後一分鐘或更長時間後會出錯，然後又切換回Safari / Chrome。
