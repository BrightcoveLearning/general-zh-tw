概述

出版商通常需要修改其媒體清單的內容特性，因為技術和業務原因。

為了滿足這些需求，Brightcove 創建了一個框架，可以應用規則，以修改從動態傳遞返回的資訊清單的內容。行為被定義為「if」條件，將觸發特定規則和一系列「則」參數，這些參數將定義如何修改清單。

如果您不熟悉此功能，請參閱概述：動態投放規則文檔。

套用規則

下列流程圖顯示傳遞規則的套用方式。

行為

第一個階段支援的行為或「if」條件包括下列各項：

  • 明確要求 -您可以在運行時傳遞給Playback API的配置ID。此 ID 對應至組態值，允許基礎服務做出由播放 API 所利用的決策，以傳回適當的回應資料。

  • 設備組 -用戶代理解析檢測到的設備類型。目前支援的群組包括：
    • 機器
    • 蘋果電視
    • iPad
    • iPhone
    • 桌上型電腦色彩
    • 桌上型火狐
    • 桌面野生動物園
    • 電視
    • iOS-其他
    • 六 7/8
    • 智慧型手機一般
    • 平板電腦-安卓
    • 平板電腦-點心
    • 未知
     
  • 地理 -請求設備的物理位置：
    • 大陸
    • 國家

規則

第一階段支援的規則，或「則」參數包括下列各項：

  • 媒體交付CDN -從配置用於給定動態交付帳戶中使用的CDN，該CDN用於交付媒體段。

  • 渲染特徵 -根據基礎媒體再現的特徵進行過濾。這些包括：
    • 最低視訊位元率
    • 最大視訊位元率
    • 最低視訊解析度
    • 最大視訊解析度
    • 音訊轉譯數目上限
    • 視訊轉譯數目上限
    • 第一個視訊位元率
    • 不連續性

範圍

規則可以在兩個層級套用：

  • 帳戶 -適用於對特定客戶帳戶的所有請求的規則
  • 具體要求 -根據特定請求調用的規則

規則將按照上述順序套用。稍後的規則將覆蓋較早的規則。

啟用功能

在傳遞規則的初始階段，所有規則將由 Brightcove 設定。請連絡您的客戶經理以了解定價，並為您的帳戶啟用此功能。

限制

使用傳遞規則時有一些已知的限制：

  • 媒體內容 -投放規則不適用於Smooth或pMP4視頻。
  • 僅音頻 -啟用服務器端廣告（SSAI）的投放規則要求同時包含視頻和音頻內容。這是一個 SSAI 限制。
