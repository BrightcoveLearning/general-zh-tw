一個

API
應用程序接口 -定義多個軟件中介之間的交互的計算接口。它定義了可以進行的調用或請求的種類，如何使它們，應該使用的數據格式，遵循的約定等。API 可以完全自訂，特定於元件，也可以根據業界標準進行設計，以確保互通性。

C

CDN
內容傳遞網絡 -代理服務器及其數據中心的地理分佈網絡。目標是透過相對於使用者的空間分配服務，以提供高可用性和效能。CDN 在 90 年代末開始存在，作為緩解互聯網性能瓶頸的手段。
CMS
內容管理系統 -用於管理數字內容創建和修改的計算機軟件。有許多 Web CMS，如 WordPress 和 Drupal，用於管理一個網站的內容，但 CMS 可以用來管理任何類型的內容。例如，視訊雲端媒體庫是媒體內容和媒體資產的 CMS，這就是為什麼用來管理該內容的主要 API 稱為 CMS API。

D

破折號
通過HTTP的動態自適應流 -自適應比特率流傳輸技術，可通過常規HTTP Web服務器在Internet上實現媒體內容的高質量流傳輸。類似於Apple的HTTP Live Streaming（ HLS）解決方案中，MPEG-DASH的工作原理是將內容分成一系列小片段，這些小片段通過HTTP進行服務。
DRM
數字版權管理 -工具或技術保護措施是一組訪問控制技術，用於限制對專有硬件和受版權保護的作品的使用。DRM 技術會嘗試控制受版權保護的作品 (例如軟體和多媒體內容) 的使用、修改和散佈，以及強制執行這些政策的裝置內的系統。

H

HLS
HTTP實時流 -由Apple Inc.開發並於2009年發布的基於HTTP的自適應比特率流通信協議。對通訊協定的支援在媒體播放器、網頁瀏覽器、行動裝置和串流媒體伺服器中都廣泛。
HTTP
超文本傳輸協議 -用於分佈式，協作的超媒體信息系統的應用程序層協議。HTTP 作為要求 — 用戶端中的回應通訊協定 — 伺服器運算模型。例如，網頁瀏覽器可能是用戶端，而在主控網站的電腦上執行的應用程式可能是伺服器。客戶端提交 HTTP 請求消息到服務器。該服務器，它提供資源，如 HTML 文件和其他內容，或代表客戶端執行其他功能，返回響應消息給客戶端。響應包含有關請求的完成狀態信息，也可能在其消息正文中包含請求的內容。

M

MP4
MPEG-4第14部分 -一種數字多媒體容器格式，最常用於存儲視頻和音頻，但也可以用於存儲其他數據，例如字幕和靜止圖像。不像HLS和短跑，MP4是作為單個文件而不是分段提供的，因此通常在希望觀眾能夠下載視頻的情況下使用。

R

休息
代表性狀態轉移 -一種軟件體系結構樣式，它定義了用於創建Web服務的一組約束。符合 REST 架構風格的 Web 服務，稱為 RESTful Web 服務，提供互聯網上計算機系統之間的互操作性。RESTful Web 服務允許請求的系統訪問和操作 Web 資源的文本表示，通過使用一個統一的和預定義的無狀態操作集。布萊特灣使用 RESTful API 來訪問平台的各個部分。
RTMP
實時消息通訊協議 -最初是Macromedia開發的專有協議，用於在Flash播放器和服務器之間通過Internet流式傳輸音頻，視頻和數據。宏媒體現在由 Adobe 擁有，它已經發布了協議規範的不完整版本供公眾使用。
